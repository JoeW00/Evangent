# Evangent UI/UX 瀏覽器測試計畫

## Context

本計畫對照 `docs/SDD.md` 中的視覺設計規格與驗收標準，透過瀏覽器自動化工具實際造訪 `http://localhost:5175/Evangent/`，逐項檢查 UI/UX 是否符合規劃。測試涵蓋桌機版（1440px）與手機版（375px）兩種視窗尺寸。

## 執行流程

1. **開啟瀏覽器** → 連接 Chrome 擴充功能
2. **建立 tab** → 導航到 `http://localhost:5175/Evangent/`
3. **桌機測試** (1440×900)：測試 1, 3, 4, 5, 6, 7, 8, 11, 12, 13
4. **手機測試** (375×812)：測試 9, 10
5. **跨狀態測試**：測試 2（需先產生閱讀記錄）
6. **產出報告**：每個測試截圖存檔，標註通過/失敗/問題描述

預計產出：一份含截圖的問題清單 Markdown 檔案，列出所有不符 SDD 的項目與建議修正方向。
