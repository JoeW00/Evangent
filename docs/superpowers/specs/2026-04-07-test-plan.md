# Evangent 測試計畫

## Context

Evangent 目前沒有任何測試。專案分兩部分：Python docx→Markdown 轉換管線，以及 React 前端閱讀器。本計畫為兩邊建立完整的測試套件，確保資料管線與前端邏輯的正確性。

---

## Step 1: Python 測試環境設定

**修改 `pyproject.toml`**：新增 pytest dev dependency
```toml
[dependency-groups]
dev = ["pytest>=8.0"]
```

**建立 `tests/conftest.py`**：共用 fixture（英文段落範例、mock docx helper）

**執行**：`uv sync && uv run pytest tests/`

---

## Step 2: Python 測試案例

### `tests/test_zh_detection.py` — 中文偵測函式

**`is_zh_section_title(text, prev_empty, next_empty)`**

| 測試案例 | 輸入 | 預期 |
|---------|------|------|
| 已知章節標題 → True | `"真實的文化"` | `True` |
| 另一個已知標題 | `"腦海中的孤兒院"` | `True` |
| 普通段落 → False | `"這是一段普通的文字"` | `False` |
| 空字串 → False | `""` | `False` |
| 純空白 → False | `"   "` | `False` |
| 前後有空白仍匹配（strip） | `"  真實的文化  "` | `True` |
| 已知標題的子字串不匹配 | `"真實的"` | `False` |
| 對話文字不誤判 | `"「你要去哪裡？」他說。"` | `False` |

**`is_scripture_ref_zh(text)`**

| 測試案例 | 輸入 | 預期 |
|---------|------|------|
| 標準經文引用 | `"詩篇 23:1"` | `True` |
| 帶中文引號 | `"「約翰福音 3:16」"` | `True` |
| 超過 60 字的段落（含書名）| 長字串 >60 chars | `False` |
| 非經文文字 | `"今天是美好的一天"` | `False` |
| 空字串 | `""` | `False` |
| 恰好 59 字的經文引用 | 建構長度 59 的字串 | `True` |

### `tests/test_en_conversion.py` — `convert_en_chapter`（最高價值）

純函式，輸入段落清單，輸出 Markdown 字串。

| 測試案例 | 說明 |
|---------|------|
| 基本段落轉換 | 3 段普通文字 → 以空行分隔輸出 |
| Part info → blockquote | `part_info="Part 1: ..."` → 輸出 `> **Part 1: ...**` |
| 無 part info 則省略 | `part_info=None` → 直接從 `# Title` 開始 |
| 標題 → H1 | 輸出含 `# {title}` |
| Running header 被過濾 | 段落中含 `"Building with God"` → 不出現在輸出 |
| 版權聲明被過濾 | Baker Publishing 文字 → 不出現在輸出 |
| Section → H2 | `sections=["Creating Culture"]` 且段落含該文字 → `## Creating Culture` |
| Prayer Prompts → H3 🙏 | `"Prayer Prompts"` → `### 🙏 Prayer Prompts` |
| 經文引用 → blockquote | `"John 3:16"` (短) → `> *John 3:16*` |
| 長經文不變 blockquote | 以 "John" 開頭但 >80 字 → 保持普通段落 |
| Ibid. → 斜體 | `"Ibid."` → `*Ibid.*` |
| 多空行被壓縮 | 連續空字串不會產生 3+ 連續換行 |

### `tests/test_zh_conversion.py` — `convert_zh_file`

用 `unittest.mock.patch("docx.Document")` mock，讓 `.paragraphs` 回傳自訂文字清單。

| 測試案例 | 說明 |
|---------|------|
| Part info → blockquote | `part_info="第一部：..."` → `> **第一部：...**` |
| 標題 → H1 | 輸出含 `# {title}` |
| 已知章節標題 → H2 | `ZH_KNOWN_SECTIONS` 內的文字 → `## {text}` |
| 禱告提示 → H3 🙏 | `"禱告提示"` → `### 🙏 禱告提示` |
| 開頭標題段落被跳過 | 含章節編號/標題/Part 關鍵字的前導段落 → 略過 |
| 普通段落正常輸出 | 一般文字不變 |
| 多空行被壓縮 | 不超過 2 個連續換行 |
| 結尾單一換行 | `result.endswith("\n")` 且非 `"\n\n"` |

## Step 3: JavaScript 測試環境設定

**安裝 devDependencies**（在 `web/` 下）：
```bash
npm install -D vitest @testing-library/react @testing-library/jest-dom jsdom
```

**修改 `web/vite.config.js`**：加入 `test` 區塊
```js
test: {
  environment: "jsdom",
  setupFiles: ["./vitest.setup.js"],
  globals: true,
}
```

**建立 `web/vitest.setup.js`**：匯入 jest-dom matchers、mock `matchMedia`

**修改 `web/package.json`**：`"test": "vitest run"`, `"test:watch": "vitest"`

**執行**：`cd web && npm test`
